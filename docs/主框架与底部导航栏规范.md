# 主框架与底部全局导航栏规范

## 1. 概述
当前应用采用移动端常见的“底部一级菜单栏（Tab Bar）”作为全局导航框架。玩家可以通过底部的导航栏在主要玩法和系统之间进行快速无缝切换。

## 2. 底部导航栏布局
位于屏幕最下方，固定悬浮，层级最高（Z-index较高），任何一级页面滑动时底部导航栏不随之滚动。

### 菜单项（从左至右）
1. **商店 (Store)**：游戏的商业化与资源兑换模块。
2. **升级 (Upgrade)**：全队/主角养成、技能强化模块。
3. **冒险 (Adventure)**：**核心主玩法**，推图、战斗、挂机收益等核心循环。
4. **装备 (Equipment)**：获取、强化、佩戴装备的系统。
5. **女仆 (Maids)**：当前所处玩法，即“女仆藏品系统”，涵盖女仆的抽取、羁绊、养成与管理。

## 3. 视觉与交互定义
- **常态状态**：灰色或暗色图标+文字。
- **选中状态**：高亮颜色（如主题色）、图标可能变大或附带动画高亮，提示当前所在模块。
- **红点提示**：当对应系统内有可领取的奖励、可强化的提示时，导航栏对应图标右上角显示小红点或数字。
- **交互过渡**：点击不同Tab时，上方主体内容区域进行页面切换，可加入淡入淡出或轻微的平移过场动画。
